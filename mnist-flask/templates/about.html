Overview

This project was created as a means of answering an all-too-common question (for me): What type of bird does a feather belong to? This is a difficult question given the range of feathers that birds have - contour feathers, semiplumes, down feathers, filoplumes, bristles, and powder down - all of which display a highly variable appearence. In this regard, Artificial Intelligence is a highly useful tool for answering this question. We're able to use a Convolutional Neural Network (CNN) to identify key features in uploaded images and match them against a library of previously learned representations to identify the species the feather came from. We rely on ResNet, a state-of-the-art architecture for visual recognition tasks, for this purpose.

Different organizations have used AI in similar applications to identify bird species from pictures, from recordings of bird calls, and through other methods. Our approach is unique in its focus on feathers alone to identify the species. This presents a unique strength, in that additional context isn't needed for identification (birdsong is difficult to capture without background noise pollution, a user may not have time to snap a picture of the bird), and gives the user a chance to learn about what species are present in an area without needing to match the temporal activity patterns exhibited by a species. At the same time this is also a limitation of the project, as some of the feathers in a given species' six types may not contain enough strong, visually distinguishable features to prevent misclassification. As such, the best results will come from a bird's contour feathers, which contain the most visually salient features, and are reletively distinct due to the role they playing giving shape and color to a bird.

This application is meant for purely educational purposes.


Implementation

The computer vision model mentioned above is the backbone of this project, and as such, the rest of the project was built around it. The model was trained and tested in Python, using Tensorflow. Interpreting model results can be tricky at best. We used Flask to stand up this service as a web application, to ensure that we deliver a UI that is accessible and quickly communicates key results without overly-involving the user in the minutiae of deep learning. We used Materialize as a front-end for the project, given its ability to extend seamlessly across mobile devices.

Several packages and databases were absolutely essential in standing up this application. The Integrated Taxonomic Information System (ITIS) provided a fantastic source of species information and was the primary source of linking scientific names with common names. After identifying the common name for a given species, Python's Google Images Download library helped to identify candidate images for each species. Finally, the Wikipedia library helped to provide links to further information for a given species.



Next Steps

This project is far from complete. There's several things I'd like to add to this project to increase its functionality or simply improve the accuracy of our predictions. These include:

  • Upgrade model from 50 species to 250 species
  • Investigate other data augmentation strategies to improve model performance
  • Implement geographic bounding to narrow the range of potential predictions
  • Include a map to record the location of finds recorded in the app
